    19. DELETING LOCAL BRANCHES

    To delete the local GIT branch we can try one of the following commands

    • The -d option stands for --delete, which would delete the local branch, only if you have already pushed and merged it with your remote branches. git branch -d branch_name
    • The -D option stands for --delete --force, which deletes the branch regardless of its push and merge status, so be careful using this one! git branch -D branch_name

    DELETING REMOTE BRANCHES

    To delete a remote branch you can use the following command:

    • git push <remote_name> --delete <branch_name>

    *Push to remote branch and delete*

    If you ever want to push your local branch to remote and delete your local, you can use git push with the -d option as an alias for --delete.

    • git push <remote_name> --delete <branch_name>

BASIC COMMANDS

  • $ git init //Initializes a new Git repository. Until you run this command inside a repository or directory, it’s just a regular folder. Only after you input this does it accept further Git commands.
  • $ git add /<"file"/> //Add File(s) To Index
  • $ git status //Check Status Of Working Tree
  • $ git commit //Commit Changes To Index
  • $ git push //Push To Remote Repository. If you’re working on your local computer, and want your commits to be visible online on GitHub as well, you “push” the changes up to GitHub with this command.
  • $ git pull //Pull Latest From Remote Repository. If you’re working on your local computer and want the most up-to-date version of your repository to work with, you “pull” the changes down from GitHub with this command.
  • $ git clone //Clone Repository Into A New Directory
  • $ git rm --cached /<"file"/> //Remove File From The Staging Area (after doing git add /<"file"/>)
  • $ git add *.html //Adds Any File That Has The HTML Status
  • $ git add . //Adds All files
  • $ git branch Login //Creates a branch called "Login"
  • $ git checkout Login //this will switch to the "Login" branch
  • $ git merge Login //this will merge the login branch to the master branch
  • $ git merge Login //this will merge the login branch to the master branch
  • $ git push -u origin master //f you are publishing a local branch for the first time on a remote, the "-u" option is helpful. It makes sure that a tracking connection between the local and the newly created remote branch is established
  • $ git remote -v //lists the current remote urls
  • $ git merge master //merging the master branch into another one e.g.feature
  • $ git rebase master //this moves the entire "feature" branch to begin on the tip of the master branch. N.B. never use git on public branches
  • $ git diff //Show changes between commits, commit and working tree, etc
  • $ git branch //Show which branch you are on
  • Merging vs Rebasing //Click here to learn how to use both
  • git log //displays all commit messages in the repo
  • git show //shows all the different files in a commit
  • git merge "branch_name" //merges the branch_name with the master branch // make sure you you are in the master branch

OPTIONS

  • --all //Pushes all local branches.
  • --tags //Pushes all local tags.
  • --delete //Deletes the specified remote branch.
  • -u //This flag creates an upstream tracking connection and is especially useful when publishing a local branch on a remote for the first time.
  • --track //This creates a new local branch with the same name as the remote one - and directly establishes a tracking connection between the two.
  • -f //this flag forces an update
  • -i //this flag makes it interactive
  • r //recursive (it's a function that calls itself i.e. a snake swallowing its tail)
